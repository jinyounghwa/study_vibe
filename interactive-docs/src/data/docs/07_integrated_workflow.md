## 7.3 실전 프로젝트 예제: 전자부품 가격 모니터링 시스템

### 프로젝트 개요

**목표**: 전자부품 가격을 모니터링하고 최저가를 알려주는 시스템

**기본 사양**:
- 사용자가 추적하고 싶은 부품 등록
- 매일 자동으로 여러 쇼핑몰 가격 조회
- 가격 인하 시 이메일 알림
- 가격 추이 그래프 시각화

### Step 1: 스펙 작성 (Web Interface)

```markdown
## 전자부품 가격 모니터링 시스템 스펙

### 주요 기능

1. 부품 추적
   - 사용자가 추적하려는 부품 추가 (부품명, 최대 허용 가격)
   - 부품 목록 조회
   - 추적 중인 부품 삭제

2. 가격 모니터링
   - 매일 자정에 자동으로 여러 쇼핑몰의 가격 조회
   - 최저 가격 기록
   - 이전 가격과 비교

3. 알림
   - 가격이 설정한 최대 가격 이하로 인하되면 이메일 알림
   - 특정 부품이 50% 이상 할인되면 푸시 알림

4. 시각화
   - 부품별 가격 추이 그래프
   - 최저가 기록 표시

### 기술 스택
- Backend: Node.js + Express + TypeScript
- Database: PostgreSQL
- Frontend: React + TypeScript + Chart.js
- Infrastructure: Docker + GitHub Actions
```

### Step 2: Claude Code로 백엔드 구현

```
단계 1: 프로젝트 초기화

Claude Code가 수행할 작업:
1. 프로젝트 구조 생성
2. 의존성 설치 (express, postgresql, typescript 등)
3. TypeScript 설정
4. 데이터베이스 스키마 정의

산출물:
- 완전한 Node.js 프로젝트 구조
- 환경 변수 템플릿
- 데이터베이스 마이그레이션 준비

---

단계 2: 데이터 모델 구현

사용자: "사용자, 부품, 가격 기록 데이터 모델을 정의하고
PostgreSQL에 마이그레이션해줘"

Claude Code가 수행할 작업:

사용자: "다음 API를 구현해줘:

- GET /api/products (부품 목록)
```

## 7.4 협업 및 코드 리뷰

### 코드 리뷰 체크리스트

```
기능 정확성:
□ 스펙의 모든 요구사항을 만족하는가?
□ 엣지 케이스를 고려했는가?
□ 테스트가 충분한가?

코드 품질:
□ 변수명이 명확한가?
□ 함수가 단일 책임을 갖는가?
□ Skills를 따르고 있는가?

성능:
□ N+1 쿼리 문제가 없는가?
□ 불필요한 연산이 없는가?
□ 메모리 누수 가능성은 없는가?

보안:
□ 입력값 검증이 충분한가?
□ SQL injection 취약점이 없는가?
□ 민감한 정보가 노출되지 않는가?
```

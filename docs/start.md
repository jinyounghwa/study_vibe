2부. Claude 개발 환경 선택 가이드
2.1 Claude 접근 방식 개요

Web/Mobile/Desktop 인터페이스 (claude.ai)
CLI 도구 (Claude Code)
Editor 통합 (Cursor, Windsurf 등)
API 직접 활용

2.2 Web/Mobile/Desktop 인터페이스 특징

장점: 즉시 사용 가능, 파일 업로드/다운로드, Artifacts 시각화
단점: 개발 환경과 분리, 복사-붙여넣기 필요
적합한 상황: 기획 단계, 스펙 작성, 프로토타이핑
주요 기능: 파일 생성, 웹 검색, Memory, Projects

2.3 CLI 도구 (Claude Code) 심층 분석

개념: 터미널에서 Claude와 직접 대화하며 코딩
장점: 프로젝트 컨텍스트 자동 인식, 파일 직접 수정, Git 통합
단점: 시각적 피드백 부족, 학습 곡선
설치 및 설정 방법
기본 명령어 및 워크플로우
적합한 상황: 백엔드 개발, 스크립트 작성, 리팩토링

2.4 Editor 통합 (Cursor, Windsurf) 심층 분석

개념: IDE 내에서 AI 지원 받기
Cursor의 특징과 강점

Inline editing, Cmd+K 단축키
Composer 모드로 멀티파일 편집
코드베이스 인덱싱 및 컨텍스트 이해


Windsurf의 특징과 강점

Cascade 모드의 자율적 작업 수행
VS Code 기반 친숙한 인터페이스


장점: 코드 편집 흐름 끊김 없음, 실시간 제안
단점: 구독 비용, 에디터 종속성
적합한 상황: 프론트엔드 개발, 복잡한 리팩토링, 실시간 코딩

2.5 상황별 최적 도구 선택 가이드

초기 기획 및 스펙 작성: Web Interface
빠른 프로토타이핑: Web Interface + Artifacts
백엔드 API 개발: Claude Code (CLI)
프론트엔드 UI 개발: Cursor/Windsurf
전체 프로젝트 개발: 혼합 전략 (단계별 도구 전환)
팀 협업: Projects 기능 활용

2.6 도구 간 워크플로우 통합

Web에서 스펙 작성 → CLI로 백엔드 구현 → Editor로 프론트엔드 작업
각 도구의 장점을 활용한 하이브리드 접근법
컨텍스트 및 코드 공유 전략

3부. 스펙 기반 개발의 핵심
3.1 좋은 스펙이란 무엇인가

명확성: 모호함 없는 요구사항
완결성: 필요한 모든 정보 포함
실행 가능성: AI가 바로 구현할 수 있는 수준

3.2 스펙 작성의 7단계 방법론

Step 1: 기능 목적과 사용자 시나리오 정의
Step 2: 데이터 구조 및 스키마 설계
Step 3: API 엔드포인트 및 인터페이스 명세
Step 4: 비즈니스 로직 플로우차트
Step 5: 에러 핸들링 및 예외 상황 정의
Step 6: 성능 요구사항 및 제약조건
Step 7: 테스트 시나리오 및 검증 기준

3.3 실전 스펙 작성 예제

간단한 기능: 사용자 인증 스펙
중간 복잡도: 게시판 CRUD 스펙
복잡한 기능: 실시간 채팅 시스템 스펙

3.4 스펙 작성 시 주의사항

기술 스택 명시의 중요성
폴더 구조 및 네이밍 컨벤션
의존성 및 라이브러리 버전 지정

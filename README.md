# Study Vibe - AI-Assisted Development Learning Guide

**Study Vibe**는 Claude와 함께 효율적인 소프트웨어 개발을 하기 위한 포괄적인 학습 가이드입니다. AI 코드 생성 도구를 활용하여 개발 생산성을 극대화하는 방법을 체계적으로 배울 수 있습니다.

이 가이드는 개인 개발자부터 팀 단위의 조직까지, 모든 규모의 개발팀이 AI와 협업하여 더 빠르고, 더 효율적으로, 더 저렴하게 소프트웨어를 개발할 수 있도록 설계되었습니다.

## 🎯 Vibe Coding이란?

**Vibe Coding**은 Claude 같은 고급 AI 모델과 협업하여 소프트웨어를 개발하는 새로운 패러다임입니다.

### 정의
Vibe Coding은 단순히 "AI가 코드를 작성하는 것"이 아닙니다. 개발자와 AI가 협업하여 더 빠르고, 더 효율적으로, 더 고품질의 소프트웨어를 함께 만드는 방식입니다. 개발자의 아이디어와 방향성, AI의 코드 생성 능력을 결합하여 최고의 결과를 만들어냅니다.

### 핵심 원칙

#### 1️⃣ **명확한 명세 (Specs)** - 품질의 기초
- 요구사항을 **상세히 정의**하면 AI가 더 정확한 결과를 생성합니다
- 불명확한 지시: "로그인 기능 만들어" → 품질 낮음
- 명확한 명세: "OAuth 2.0을 사용한 로그인, JWT 토큰, 토큰 TTL 1시간" → 품질 높음
- **명세가 명확할수록 반복 수정이 줄어들고 첫 시도 성공률이 높아집니다**

#### 2️⃣ **반복적 협업** - 점진적 개선
- AI는 완벽하지 않으므로 여러 번의 상호작용 필요
- 워크플로우:
  ```
  명확한 명세 작성
    ↓
  AI에게 첫 구현 요청
    ↓
  코드 검토 및 테스트
    ↓
  문제 발견 시 피드백
    ↓
  개선 요청 및 재구현
    ↓
  완성
  ```
- 이 과정에서 개발자와 AI가 함께 품질을 높여갑니다

#### 3️⃣ **도구 최적화** - 각 상황에 맞는 도구 선택
- 각 개발 단계와 작업 유형에 맞는 도구를 선택합니다:
  - **계획 단계**: Claude Web Interface (대화식 사고)
  - **백엔드 개발**: Claude Code CLI (빠르고 효율적)
  - **프론트엔드 개발**: Cursor IDE (IDE 통합)
  - **UI 프로토타입**: v0.dev (빠른 시각화)
  - **데이터 통합**: MCP를 통한 웹 스크래핑 및 API 연동
- **한 가지 도구만 고집하기보다는 상황에 맞는 도구를 조합**하여 사용합니다

#### 4️⃣ **생산성 향상** - 측정 가능한 효과
- 반복적인 작업 자동화로 **3-5배 생산성 향상**
- 보일러플레이트 코드 자동 생성
- CRUD 작업 자동화
- 반복적인 리팩토링 신속화
- **예시**:
  - 전통: API 엔드포인트 구현에 1일 소요
  - Vibe Coding: 30분 + 30분 검토 = 1시간 (6배 향상)

### Vibe Coding의 3단계 사이클

```
┌─────────────────────────────────────────────────┐
│ 1. 명세 작성 (15%)                               │
│    ├─ 목표 정의                                 │
│    ├─ 요구사항 나열                             │
│    ├─ 제약조건 정의                             │
│    └─ 예제 작성                                 │
├─────────────────────────────────────────────────┤
│ 2. AI 협업 (65%)                                │
│    ├─ Claude에 명세 전달                       │
│    ├─ 코드 생성                                 │
│    ├─ 첫 검토                                   │
│    └─ 필요시 개선 요청                         │
├─────────────────────────────────────────────────┤
│ 3. 반복 개선 (20%)                              │
│    ├─ 버그 수정                                 │
```

## 💬 자주 묻는 질문 (FAQ)

### 기본 개념
**Q: Claude와 Copilot의 차이점은?**
A: Claude는 문맥 이해와 복잡한 추론에 우수합니다. Copilot은 IDE 통합이 뛰어나고 빠릅니다. Vibe Coding은 Claude 기반의 깊은 협업 방식이고, Copilot은 자동완성 보조 도구입니다.

**Q: Vibe Coding을 하려면 개발 경험이 필요한가?**
A: 네, 기본적인 프로그래밍 경험 (1년 이상)이 있으면 좋습니다. AI가 코드를 작성하지만, 명세 작성과 검증은 개발자 몫입니다.

**Q: 보안에 민감한 프로젝트는 어떻게?**
A: 민감한 코드는 로컬에서만 처리합니다. 외부 API 연동이 필요한 부분만 Claude에 요청하고, 핵심 비즈니스 로직은 직접 검토합니다.

### 비용
**Q: 월 비용이 정말 $5-10인가?**
A: Claude API는 사용량 기반 과금입니다.
- Haiku (저비용): 월 $5-10
- Sonnet (균형): 월 $20-50
- Opus (고품질): 월 $50-100
IDE 도구 (Cursor 등)는 별도 비용입니다.

**Q: 비용을 더 절감할 수 있나?**
A:
- Claude Haiku 사용 (3배 저렴)
- 캐싱 활용 (컨텍스트 비용 60% 감소)
- 배치 처리 (50% 할인)

### 도구 선택
**Q: Claude Code를 쓸까, Cursor를 쓸까?**
A:
- **Claude Code**: 백엔드, 빠른 개발, CLI 환경
- **Cursor**: 프론트엔드, IDE 통합, 시각적 피드백
→ 둘 다 사용하세요! 각각의 장점이 다릅니다.

**Q: v0.dev는 언제 사용?**
A: UI 프로토타입과 디자인 검증이 필요할 때. 30분 안에 전문적 UI를 만들 수 있습니다.

### 성능
**Q: AI가 생성한 코드가 느리진 않나?**
A: 평균적으로 좋습니다. 성능이 중요한 부분은 명세에서 명시하세요:
```
API 응답 시간: < 500ms (P99)
메모리 사용량: < 100MB
```

**Q: 버그가 많진 않나?**
A: 명확한 명세로 70-80% 첫 시도 성공률을 얻습니다. 테스트 작성 후 버그는 50% 감소합니다.

### 팀 도입
**Q: 팀에 어떻게 도입할까?**
A:
1. 1-2명 파일럿 (1주)
2. 전체 팀 교육 (1주)
3. 첫 프로젝트 함께 시작 (1주)
4. 지속적 개선

**Q: 시니어 개발자가 항상 검토해야 하나?**
A:
- 초기: 모든 코드 검토 (1-2주)
- 안정화: 30% 샘플링 검토 (이후)
